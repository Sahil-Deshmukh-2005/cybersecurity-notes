## Room-5 : Cloud Computing Fundamentals

***

You had a funtastic idea of developing an app, you host it on your own machine in your own country.  
But users in other country faces lag.  
Your machine shuts down.  
These were the limitations of using your own machine to host a service.  
That's when `Cloud Computing` came in play.  
`Cloud` is built on top of technologies like virtualization and containers.  
The `Cloud` lets you use the computing resources over the internet. This makes your application easier to access and more reliable.

### Cloud Benefits and Characteristics
- Scalability = Easily scale up or down as your application's need changes.  
- On-demand Self-service = Create or Remove servers and storage instantly without waiting for hardware.
- Pay only for what you use = You are charged based on usage, no upfront cost.
- Security = Cloud providers protect the infrastructure with strong security measures.
- High Availability = Applications keep running even if the parts of system fails.
- Global Access = Your application can be accessed by users anywhere in the world.

***

### Types of Clouds
#### Deployment Types :- 
1) Public Cloud =  
Used by startups, websites and global apps because it is affordable, easy to scale, and requires no infrastructure managemant.
`Public Cloud` services are preferable for nearly every use case.
  
2) Private Cloud =  
Used by banks, healthcare and government organizations because it offers great control, customization and compliance for sensitive data.
  
3) Hybrid Cloud =  
Used by companies like e-commerce platforms that need to keep sensitive data private while still scalling publically during the high demand.

#### Different ways to use Cloud Services :-
1) Infrastructure as a Service (IaaS) =  
You rent basic computing resources such as virtual servers, storage, and networking.  
You are responsible for managing the `OS` and you application, while the provider manages the physical hardware.

2) Platform as a Service (PaaS) =  
The `Cloud` provider manages the infrastructure and the `OS`.
You focus on building, deploying and running your application without worring about servers.

3) Software as a Service (SaaS) =  
You use the complete application over the internet.
The provider manages everything and you access the software through a browser or app.
Ex, Gmail or Zoom

### Basic CLoud Terminologies (AWS)
- EC2 (Virtual Machine/ Server)  
  `EC2` represents a virtual computer in `Cloud`. Just like a real computer, it has a CPU, and memory (RAM) and can run applications.
  Whenever you add an `EC2` instance, you are adding a computer to the environment.

- Instance Type (Ex, t2, t3, m5)  
  `Instance Types` describes how powerful the virtual computer is. Some have more CPU and RAM and are therefore more expensive.
  You choose the `Instance Type` based on your needs, knowing that:
  - Bigger Instance = More power + Higher cost
  - Minor Instance = Less power + Lower cost
